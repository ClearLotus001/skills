# 故障排查

## 1. rules.json 编译失败
- 检查 JSON 语法是否正确。
- 检查顶层必需字段是否完整。
- 检查 relation_rules 中引用的数据集是否存在。
- 检查选择的 rule_set 是否存在。

## 2. 解析阶段告警
- 安装缺失解析依赖：
- `.xlsx/.xlsm` 需要 `openpyxl`
- `.xls` 需要 `xlrd`
- `.xlsb` 需要 `pyxlsb`
- `.csv` 需确认编码格式。

## 3. 续跑异常
- 检查 `run_state.json` 是否存在且结构有效。
- 检查输入哈希与规则哈希是否发生变化。

## 4. 性能瓶颈
- 缩小激活 rule_set 范围。
- 在内存允许前提下调整 chunk 大小。
- 将超大任务拆分为独立批次运行。

## 5. 报告不一致
- 确认输入、规则与执行参数保持一致。
- 用相同 run profile 复跑验证。
- 对比 issue_id 是否稳定。
