# 故障排查指南

## 1. `rules.json` 编译失败

常见表现：
- 日志出现 `compile_rules` 失败
- 流程在预检查阶段退出

排查步骤：
1. 检查 JSON 语法是否合法。
2. 检查顶层是否包含 `datasets`。
3. 检查规则里引用的数据集是否都在 `datasets` 中定义。
4. 若使用 `--rule-set`，确认该分组存在且包含有效 `rule_id`。

## 2. 解析阶段告警或失败

常见表现：
- 日志提示解析告警并中止（默认严格模式）
- Excel 文件无法打开或读取异常

排查步骤：
1. 查看 `<out>/_run.log` 具体告警。
2. 安装依赖：`pip install -r requirements.txt`。
3. 对 Excel：确认文件未损坏、未被占用。
4. 对 CSV/TSV：检查编码并转为 UTF-8。

## 3. 包结构预检告警

常见表现：
- 日志包含“包结构预检”字样
- 提示关键部件缺失、关系目标不存在或 XML 解析失败

处理建议：
1. 先在 Excel 中另存为新文件后重试（可修复部分轻微包结构问题）。
2. 若是手工处理过压缩包，检查 `[Content_Types].xml`、`_rels/.rels`、`xl/workbook.xml` 是否完整。
3. 若确认源文件可用且仅需快速继续，可临时加 `--skip-xlsx-package-check` 跳过预检。

## 4. 公式相关告警

当前主链路：
- 发现公式单元格后会执行内置的 LibreOffice 公式重算流程。
- 若重算失败或重算后仍存在 Excel 错误码，会产生解析告警。

处理建议：
1. 手工在 Excel/LibreOffice 重新计算并保存后重试。
2. 检查公式是否存在 `#REF!`、`#DIV/0!` 等错误。
3. 确认本机 `soffice`（LibreOffice）可执行。

## 5. `--resume` 未按预期生效

排查步骤：
1. 确认输出目录存在 `run_state.json`。
2. 确认输入文件与规则未变化（哈希变化会触发重新执行）。
3. 查看日志中状态加载信息。

建议：
- 输入变更较大时，删除旧输出目录后全量重跑。

## 6. 性能或内存问题

优化建议：
- 通过 `--rule-set` 先跑关键规则。
- 适当调小 `--chunk-size`（如 `500`）。
- 超大数据集按批次拆分执行。

## 7. 文件找不到

排查步骤：
1. 核对 `--inputs` 路径。
2. 核对 `datasets` 中 `file` / `file_pattern` / `file_path`。
3. 先执行 `--scan`，确认文件是否被发现。

补充说明：
- 未提供 `--inputs` 时，流程不会执行，会先提示补充输入路径。
- 未提供 `--out` 时，会自动创建临时输出目录并在日志中打印该路径。

## 8. 结果不一致

排查步骤：
1. 核对输入文件 SHA-256 是否一致。
2. 核对规则文件和执行参数是否一致。
3. 避免在规则表达式中引入不稳定逻辑。

## 9. 依赖缺失

```bash
pip install -r requirements.txt
```

核心依赖：
- `openpyxl`
- Python 标准库
